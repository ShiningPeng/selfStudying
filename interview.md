# 看一本前端面试书籍笔记
## 第一章HTML
1. 谈谈你对Web标准以及W3C的理解和认识
  - 标签要闭合，英文小写，且不要嵌套混乱，用标签语义化来提高搜索的效率。
  - 使用外链式的CSS和JS脚本，使结构、样式、行为分离，内容能被更广泛的设备所访问
  - 代码精简，开发组件化，代码易维护、可复用，改变、升级方便
2. HTML 和 XHTML 有什么区别
  - HTML是一种基于Web网页的设计语言，XHTML是一种基于XML、语法严格、标准的设计语言
  - 不同的是XHTML元素必须正确的嵌套，元素必须闭合，标签必须小写，必须有根元素，而HTML没有这些限制
3. 严格模式和混杂模式如何区分？如何触发这两种模式？
  - 严格模式是一种要求严格的DTD，不允许使用任何表现层的语法；混杂模式是一种向后兼容的解析方法
4. 什么是静态网页？什么是动态网页？
  - 静态网页是指没有数据交换的网页，没有数据库参与，没有服务器端数据的加载，如只有HTML+CSS+JS做的网页
  - 动态网页是指有后台数据参与的网页，网页中的数据是从数据库中提取的，需要有后台逻辑的支持。如JSP页面和ASP页面
5. DOCTYPE 有什么作用？区分严格模式和混杂模式有何意义？
  - 告诉浏览器用什么类型规范来解析这个文档，DOCTYPE不存在或格式不正确都会导致文档以混杂模式呈现。
  - 严格模式的排版和JS运行模式都是以浏览器的最高标准运行。混杂模式页面以宽松的向后兼容的方式显示。
6. 如何调式网页代码？如何查看网页源代码？
  - F12，右键查看网页源代码
7. 语义化的主要目的是什么？
  - 让页面的内容结构化，便于浏览器解析和搜索引擎解析，并提高代码的可维护性和可重用性。
8. 锚点的作用是什么？如何创建锚点？
  - 类似于书签，用于链接到文档中的某个位置。
  - 创建时可用name属性为其命名
9. 列举常用的结构标签，并描述其作用。
  - <header><section><nav><article><footer><aside>
10. 超级链接有哪些常见的表现形式？
  - 普通超链接、下载链接、电子邮件链接、空链接、锚点跳转、用于实现特定的代码功能
11. 你测试过那些浏览器的页面？他们的内核分别是什么？
12. div是什么？在div出现之前用什么做网页布局？
  - div是网站布局的盒子标签，div出现之前是使用table布局。因为table布局嵌套很多，网站加载慢（table无法局部渲染），布局层级不清晰。
13. img标签上的title和alt属性的区别是什么？
  - title是光标悬浮在标签上时显示信息，而alt是图片加载失败时用文字替代
14. 空元素（单标签）有哪些？
  - 空元素也就是所谓的单标签，常见的有<br><hr><img><input><link><meta>
15. 简述一下src 和 href 的区别。
  - src表示来源地址，用在img，script、iframe等元素上，href（hypertext reference）用在link和a等元素上。src是引入，src是页面不可少的一部分，href是引用，只是有关联。
16. 简述一下<strong><em>和<b><i>标签的区别。
17. 你知道多少种DOCTYPE？HTML与XHTML有什么共同点？
  - 3种：Strict、Transitional 和 Frameset
18. 在新的窗口打开链接的方式是什么？
  - target=_blank
19. HTML、CSS、JavaScript的关系是什么？
  - 简单地说，HTML是结构如文字图片视频等，CSS是对结构进行包装，比如字体颜色边框，改变外观的东西，JS是用来实现网页上的特效和交互。
20. 在一个特定的框架中如何使用HTML中的超链接定位？
  - 使用target属性在指定的框架中打开被链接的文档。
21. 说说iframe有哪些优点？
22. 请说说iframe有哪些缺点？
23. 说说你对语义化的理解。
  - 语义化标签特点：
    1. 丢失样式时能够让页面呈现出清晰的结构
    2. SEO是指和搜索引擎建立良好的沟通，有助于爬虫抓取更多的有效信息。爬虫依赖标签来确定上下文和各个关键字的权重。
    3. 其他设备解析（如屏幕阅读器，盲人阅读器，移动设备）以有意义的方式来渲染网页。
    4. 在团队开发和维护中，语义化更具可读性，是未来网页发展的重要方向。遵循W3C标准的团队开发的网页，都会遵循这个标准，以减少差异。

2. 列举几种DOM节点

3. srcipt标签中defer 和 async 属性的区别
  - defer 属性规定是否延迟执行脚本，直到页面加载为止，async 属性规定脚本一旦可用，就异步执行
  - defer并行加载js文件，会按照页面上script标签的顺序执行。async并行加载js文件，下载完成立即执行，不会按照页面上script标签的顺序执行。
4. 闭包的理解
  - 使用闭包主要是为了设计私有的方法和变量，闭包的优点是可以避免全局变量的污染；缺点是闭包会常驻内存，增加内存使用量，使用不当很容易造成内存泄漏。在js中，函数即闭包，只有函数才会产生作用域。
  - 闭包的特性
    1. 函数嵌套函数
    2. 在函数内部可以引用外部的参数和变量
    3. 参数和变量不会被垃圾回收机制回收
5. unshift()方法
  - 在数组的最前面插入元素
  ```js
  var name = ['zhngsn'];name.unshift('lisi');
  //['lisi', 'zhngsn']
  ```
